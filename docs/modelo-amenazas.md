# Modelo de amenazas (resumido)

Qué riesgo concreto cubre cada capa, traducido a impacto de negocio para una pyme.

| Capa | Amenaza que mitiga | Impacto para la pyme si se materializa |
|---|---|---|
| 1 · Dominio y DNS | Pérdida o secuestro del dominio | Caída de web y correo; recuperación cara o imposible |
| 2 · Correo | Suplantación del dominio en correos a clientes | Fraude del CEO, facturas falsas, daño reputacional |
| 3 · Web | Compromiso vía software desactualizado; robo de datos en pago (Magecart) | Brecha de datos, obligación de notificar a la AEPD, pérdida de confianza |
| 4 · Credenciales | Reutilización de contraseñas filtradas | Acceso no autorizado a sistemas del negocio |
| 5 · Suplantación | Webs y dominios clónicos, perfiles falsos | Phishing a clientes, pérdida de ventas, daño de marca |
| 6 · Analítica | Decisiones a ciegas; tratamiento ilícito de datos | Inversión de marketing sin retorno medible; sanción AEPD |
| 7 · Continuidad | Dependencia de terceros, servicios huérfanos | Pérdida de control del negocio digital ante un conflicto o un cese |

## Perfil de atacante considerado

El modelo asume principalmente **atacantes oportunistas y automatizados** (bots que escanean en masa buscando software desactualizado, campañas de phishing genéricas, squatters de dominios), que son la amenaza real y cotidiana para una pyme, frente al atacante dirigido y sofisticado, menos probable en este segmento.

La mayoría de incidentes en pymes no son ataques selectivos: son la consecuencia de estar en la lista equivocada en el momento equivocado, con algo sin actualizar.
